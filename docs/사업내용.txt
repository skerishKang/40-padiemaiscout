## **GrantScout 프로젝트 아이템 기획서 (요약본)**

**1. 프로젝트 명칭:** GrantScout (PoC 단계)

**2. 해결하고자 하는 문제:**

*   수많은 지원사업 공고 속에서 기업/개인에게 적합한 사업을 찾고 내용을 파악하는 데 많은 시간과 노력이 소요됨.
*   공고문(주로 PDF)의 비정형 텍스트에서 핵심 정보(대상, 기간, 금액 등)를 추출하고 자사와의 적합성을 판단하기 어려움.
*   새로운 공고를 놓치거나 마감일을 관리하기 번거로움.

**3. 제안 솔루션 (GrantScout):**

*   **AI 기반 지원사업 분석 및 맞춤 추천 시스템**
*   사용자가 지원사업 공고 파일(PDF)을 업로드하거나, 향후 시스템이 자동으로 공고를 수집하면(RSS, 웹 스크레이핑 등), **Gemini AI**가 공고 내용을 분석하여 핵심 정보를 **구조화된 데이터(JSON)**로 추출.
*   추출된 정보(사업명, 지원 대상, **마감일**, 금액 등)를 사용자 친화적인 인터페이스(표 등)로 제공.
*   사용자가 입력한 **회사 정보**와 공고 내용을 비교하여 **AI가 적합성 점수 및 사유를 분석**하여 제공.
*   **(궁극적 목표)** 자동화 플랫폼(n8n, Make 등)과 연동하여, 신규 공고 발생 시 자동으로 분석하고 조건에 맞는 사용자(회사)에게 **선제적으로 알림** 발송.

**4. 주요 기능 (PoC 범위 및 확장 계획):**

*   **핵심 분석 파이프라인 (현재 구현 완료 및 안정화 단계):**
    *   [완료] Flutter 웹 앱 기반 PDF 파일 업로드 기능.
    *   [완료] Firebase Storage 저장 및 Cloud Function 트리거.
    *   [완료] Cloud Function (Node.js v2): PDF 다운로드 -> Gemini API (Flash) 호출 -> 핵심 정보 JSON 추출 (마감일 포함).
    *   [완료] Firestore 저장: `analysisStatus` (상태), `analysisResult` (JSON 결과), `extractedTextRaw` (원본 응답), `analyzedAt` (분석 시간) 등 저장.
    *   [완료] Flutter UI: 업로드 목록 표시, 분석 결과(JSON)를 표(DataTable) 형태로 표시.
    *   [진행중] Flutter UI: 분석 중 로딩 상태 표시 (`CircularProgressIndicator`, "분석 중..."), 분석 완료 시 결과 자동 표시.
    *   [진행중] Flutter UI: "원본 Gemini 응답 보기" 기능 (디버깅 및 검증용).

*   **회사 정보 및 적합성 분석 (다음 단계):**
    *   [구현 예정] Flutter 앱: 회사 정보 관리 화면 (프로필 입력/수정) 및 Firestore 저장 (`user_profiles/{uid}`).
    *   [구현 예정] Cloud Function (`checkSuitability`): (회사 정보 + `analysisResult`) 입력 -> Gemini API 호출 (시스템 프롬프트 활용) -> 적합성 점수 및 사유 반환.
    *   [구현 예정] Flutter UI: 적합성 분석 요청 버튼, 분석 결과(점수, 사유) 표시 (다이얼로그 등).

*   **사용성 개선 및 기능 확장 (PoC 이후 또는 병행):**
    *   **[구현 예정] 마감일 관리 기능:**
        *   Cloud Function: Gemini 추출 날짜(`deadline`)를 Firestore `Timestamp`(`deadlineTimestamp`)로 변환 저장.
        *   Flutter UI: 현재 날짜와 비교하여 "마감 D-N", "오늘 마감", "마감일 지남 (N일 경과)" 등 상태 표시. (색상 강조 등 시각화)
    *   [확장 고려] HWP, 웹 페이지 링크, ZIP 파일 등 다양한 형식의 공고 처리 방안 연구 (외부 솔루션 연동 등).
    *   [확장 고려] Gemini 프롬프트 지속적 튜닝 (정보 추출 정확도 향상, 적합성 분석 품질 개선).
    *   [확장 고려] UI/UX 개선: 필터링, 정렬, 검색 기능 등.

*   **자동화 및 알림 시스템 (장기 비전):**
    *   [구현 구상] RSS 피드, 웹 스크레이핑 등으로 신규 지원사업 공고 자동 수집.
    *   [구현 구상] 자동화 플랫폼(n8n, Make 등) 연동: 새 공고 발생 시 GrantScout Cloud Function(분석 API) 호출.
    *   [구현 구상] 자동화 플랫폼: 분석 결과와 등록된 회사 정보 매칭 로직 수행 (필요시 `checkSuitability` API 호출).
    *   [구현 구상] 자동화 플랫폼: 조건 일치 시 해당 사용자에게 이메일, 슬랙, 앱 푸시 등 알림 발송.

**5. 타겟 사용자:**

*   정부 지원사업을 찾는 스타트업, 중소기업 담당자.
*   연구 과제 등을 찾는 연구원, 대학생.
*   고객의 지원사업 신청을 돕는 컨설턴트.

**6. 기술 스택 (현재):**

*   **프론트엔드:** Flutter (Web)
*   **백엔드:** Firebase (Authentication, Firestore, Storage, Cloud Functions v2 - Node.js)
*   **AI:** Google Gemini API (Flash model)

**7. 현재 상태:**

*   UX 개선 작업(로딩 상태 표시, 자동 결과 표시) 진행 중.
